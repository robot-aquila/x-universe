Контроллер безопасности

Основная задача контроллера безопасности - это предотвращение контакта с
вражескими кораблями. В отличии от многих других программ, контроллер
безопасности не управляет кораблем во время боя, он просто не допускает
ситуации, когда вражеский корабль приблизится на расстояние выстрела.
С помощью прыжкового двигателя, контроллер безопасности направляет корабль на
ближайшую станцию в соседнем секторе, отправляя игроку сообщение, в котором
указана информация о секторе и атакующем корабле. По прибытии на станцию,
корабль ожидает до тех пор, пока атакующий не покинет сектор или не будет
уничтожен после чего, возобновляет выполнение предыдущего задания. Пилот
не сразу возвращается к предыдущему заданию. Некоторое время он проводит
в баре на станции, пытаясь отойти от пережитого страха. В ваших интересах
поддерживать безопасность основных маршрутов, что бы избежать задержек.

После установки программы в меню дополнительных команд кораблей, оборудованных
прыжковым двигателем, появляется пункт "SC: Контроллер безопасности", которая
инициирует запуск программы. При запуске требует указать количество батареек,
зарезервированных для прыжкового двигателя.

--------------------------------------------------------------------------------
Интерфейс: русский, английский
Совместим с XTM
Использованные команды: COMMAND_TYPE_SHIP_56

Установка:
1. Скопировать файлы архива в папку с игрой

Удаление:
1. Остановить программу на каждом корабле где она была запущена
2. Удалить ранее скопированные файлы

================
* Переработан алгоритм поиска врагов. Для каждого типа кораблей установлена
минимальная дистанция: от 3км для истрбителей до 13км для всех остальных.
Если вражеский корабль ближе указанной дистанции, то включается процедура
спасения корабля от опасности.
================
* Добавлена обработка частного случая, когда вражеские корабли
обнаружены во время нахождения корабля в пристыкованном к станции состоянии.
Процедура спасения в такой ситуации заключается в блокировании корабля на
станции.
================
* Поиск станций в соседнем секторе выполнять только есть прыжковый двигатель и
батарейки для прыжка. Иначе искать ближайшую станцию.
================
* sc-is01 При запуске контроллера запрашивается количество батареек под
прыжковый двигатель. В обычном режиме (не тревожном) контроллер периодически
проверяет состояние корабля и отправляется на дозаправку при следующих условиях:
1) В данный момент не выполняется процедура дозаправки;
2) На корабле установлен прыжковый двигатель;
3) Установленный резерв под батарейки достаточен для прыжка в соседний сектор;
4) Батареек в трюме меньше, чем требуется для прыжка в соседний сектор;
5) Свободного места в трюме больше или равно, чем требуется под батарейки для
   прыжка в соседний сектор;
6) На счете корабля есть достаточно средств для докупки батареек по максимальной
   цене до зарезервированного объема.
7) В текущем секторе есть доступный продавец батареек с количеством товара на
   продажу не менее, чем требуется для прыжка в соседний сектор;

В случае если все условия выполнены, контроллер запускает процедуру дозаправки,
которая перехватывает управление кораблем, направляет его на станцию и пытается
докупить батарейки. После этого, корабль возвращается к первоначальной
программе. Процедура дозаправки запускается с меньшим приоритетом, чем
процедура спасения корабля. Это означает, что контроллер безопасности сработает
даже в процессе дозаправки.

Журналирование процесса перезаправки позволяет отслеживать проверки процедуры
дозаправки. Эта информация может быть полезна для анализа маршрутов. Для
включения журналирования необходимо указать отрицательное значение резерва
при запуске контроллера (само значение будет использовано по модулю). В журнал
сохраняется общая информация: время проверки, мизур до следующей проверки,
код результата, сектор и название корабля. Коды результата расшифровываются
следующим образом:
0 - корабль уже находится в режиме дозаправки
1 - не установлен прыжковый двигатель
2 - резерв под двигатель меньше, чем требуется для прыжка в соседний сектор
3 - в трюме достаточно батарей для прыжка в соседний сектор
4 - в трюме нет места для батарей даже на один прыжок
5 - недостаточно денег для покупки необходимого количества батарей по макс. цене
6 - в секторе нет станции, на которой можно загрузить батарей хотя бы на прыжок
7 - запущена процедура дозаправки
